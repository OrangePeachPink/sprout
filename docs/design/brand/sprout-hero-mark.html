Sprout hero mark

The living mark for the FD-1 front-door hero — a design input to #1071. DesignQA authors it; DX consumes it verbatim; it also becomes FD-7's teaser export source.

Canonical thriving geometry from components/sprout-mark.js; motion mirrors components/sprout-motion.css (sway, 6.5 s). It's a single self-contained .svg — the CSS animates in the browser with no script and no external runtime, whether inlined into the hub or used as an <img> — so it satisfies FD-1's zero-external-runtime rule. It honors prefers-reduced-motion, and its greens read on both light and soil grounds.

How DX consumes it (FD-1 hero)

Two equivalent paths — both zero-external-runtime:

  • Inline, self-contained (recommended): paste the whole sprout-hero-mark.svg markup into the hero. Its scoped <style> carries the motion, so nothing else is needed. Size it with CSS on the <svg> (e.g. width: 148px).
  • Canonical-motion variant: if the hub already links design/components/sprout-motion.css (a local file — still zero external runtime), drop the inline <style> and add class="sprout-sway" to the foliage <g> instead. Same result, single motion source.

Motion vocabulary (per sprout-motion.css / BRAND.md §3): sway = calm idle (the hero default) · breathe = alive & well · bob = speaking/greeting · droop = thirsty. The hero ships sway; any state is a one-class swap.

Why sway, for the hero

A persistent hero element wants presence without fidget: sway is the brand's calm-idle default — alive, never busy. bob (greeting) and breathe (thriving) are on hand if a moment calls for them, but the ambient first impression is a seedling that simply, quietly lives.
