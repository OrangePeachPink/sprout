# Retro — v0.8.0 "Predict: Sprout Sees It Coming" (2026-07-21)

The first per-release retro of record. Working session: maintainer + a retrospective/prompt
agent, reviewed and extended by Workflow; every process decision below was three-way ratified
2026-07-21 and landed as the ADR-0003 amendment in the same PR as this document. Findings were
derived from **shipped** work across v0.7.1 → v0.8.0, deliberately not from the open queue
(which is skewed by intentionally parked hardware work).

## Ledger (by query — see the method note at the end)

- **141 PRs merged** since the v0.7.3 tag (2.5-day cycle) · **91 rode the accelerated (V2) lane**
- **1 deliberate revert** — a too-early retirement pull restored so retiring surfaces could stand
  beside their replacements (the confirmation cycle working), doubling as the live no-regrets
  rollback drill
- **1 red main** in 541 runs (`e979be7` — the S3 "non-blocking" job failing the run; wiring
  regression filed #1439) · **1 red release run** (the immutability/sign-order collision, #1438,
  v0.8.0 accepted asset-less)
- **80 issues closed** in the milestone · 4th external contributor merged same-day

## What worked (keep and name)

- **V2 velocity is real** — review is not the bottleneck; issue throughput is.
- **Cadence is stable** — 144 → 81 → 70 → 74 issues across the last four releases; we know our size.
- **Hardware staging is deliberate** — bench work batches into hardware releases; software
  releases stay software-only. Aged `needs:hardware` numbers are the plan, not neglect.
- **Tail-parking, now a named standard** — structure the release so it closes on maintainer-only
  work (bench steps, cut, notes); no lane blocks the close.
- **The bench pairing model** — one lane guiding hands-on, one as parallel validator, the
  maintainer's domain memory as the third instrument (five of the bench's self-corrections came
  from her knowledge, not more compute).

## What missed

- **v0.8.0 published asset-less** (#1438) — two protections collided (immutable releases ×
  post-publish signing); root cause is checklist *ordering*.
- **The falsehood family** — #1428 (Monitor said stopped while logging), #1429 (the chart's
  silent stale window), #1432 (19 plants on the Home), #1435 (/trial empty against live bands):
  one shape, a locally-correct layer confidently asserting a claim nobody walked end-to-end.
  Process answer: the **seam register** + cross-seam claims never fast-pathed (ADR-0003 §8a).
- **AC-evidence mapping drifted** from ADR-0003 §8's own requirement.
- **ADR sprawl** — minting discipline slipped (at ADR-0038); amend-by-default now enforced by a
  PR-template question.
- **Label taxonomy drifted** — `area:`/`layer:` vestigial; `for:` silently meant both owner and
  consulted; velocity labels applied to 7 of 69 open issues.
- **Checklists crept back over native sub-issues**, contrary to ADR-0003 §2.
- **Closeout ledgers were reported from impression** — including twice by the gate in this very
  window (reverts, red mains), and once by the retro draft itself (a `--limit 60` ceiling reported
  as "~60 PRs"). All corrected by query; the ledger-by-query rule is now ADR-0003 §11.

## The decisions (ratified — the operative text lives in ADR-0003)

Owner/consulted label semantics + the four planning attributes as Project fields with `just`
wrappers (§5) · the readiness gate with saved-view enforcement (§5) · evidence per-AC, the fast
path with the cross-seam carve-out, the seam register, pre-approved-scope V2, the
Workflow-self-certification capability boundary (§8a) · the three-way discovery rule (§8b) ·
commit-type set + types-only enforcement, the kickoff relay, receipts-not-content, idle-declared
(§9) · ADR amend-by-default + the template question (§10) · the retro metric / friction gauge
(§11).

## Watch for next retro

- The **friction gauge** (merge-queue arrivals vs. genuine maintainer input) — definitions fixed
  at the v0.8.1 closeout; the dial that watches both the conservative seam-register start and
  the V1-flood risk.
- Whether the `epic` label retires (pending the parent-field view verification in the board
  reshape).
- Mid-release lane-resumption (scheduled pokes) — declare-idle helps the maintainer notice
  faster, not avoid noticing; only partially addressed this cycle.
